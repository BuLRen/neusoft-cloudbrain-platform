## 集成边界

### 设计原则

本工作流与 `WF-01_AI生成月度排班` 保持一致，只承担 **AI 理解、推理、校验、输出结构化结果** 的职责。

本版本进一步约束为：**Coze 不依赖任何内部查询能力**。谁调用 Coze 工作流 URL，谁就必须一次性传入完整业务数据；Coze 只基于传入数据完成处理并返回结果。

### Coze 负责

- 解析医生请假文本
- 生成替班建议
- 校验替班建议
- 返回结构化结果给调用方

### 调用方负责

- 在调用 Coze 前准备完整输入数据
- 将医生信息、原排班、候选替班医生、业务规则一次性传入 Coze
- 接收 Coze 返回结果
- 根据 Coze 返回结果决定是否保存请假申请、创建调班申请、发起人工审核、发送通知
- 如需落库或通知，全部由调用方自行处理

### 本版本的关键约束

- Coze 不调用 `physicianService.getById(...)`
- Coze 不调用 `scheduleService.getById(...)`
- Coze 不调用 `scheduleService.findSubstitutes(...)`
- Coze 不动态查询规则配置
- Coze 只消费调用方传入的 JSON 数据

### 本工作流不包含

- 用户输入触发
- 用户确认节点
- Coze 直连业务接口
- Coze 直接落库
- Coze 直接发送通知
- Coze 直接修改排班

### 系统提示词

```markdown
# 角色定义
你是医院排班系统的请假解析助手，负责将医生自然语言请假文本转换为结构化数据。

## 你的任务
请结合请假文本和原排班信息，提取以下字段：
1. leave_date：请假日期（YYYY-MM-DD）
2. time_slot：请假时段（上午/下午/全天）
3. leave_type：请假类型（事假/病假/公假/其他）
4. reason_summary：请假原因摘要（10字内）
5. confidence：解析置信度（0-1）
6. need_manual_review：是否建议人工审核
7. parse_comment：一句话说明

## 解析规则
### 日期
- 能明确解析时，输出具体日期
- “下周三”“明天下午”等相对日期需要推算为具体日期
- 如果仍存在歧义，给出最可能结果，并将 need_manual_review 设为 true

### 时段
- 明确提到“上午/下午/全天”时直接输出
- 未明确时优先参考 original_schedule.time_slot
- 若仍无法确认，默认“上午”，并设 need_manual_review 为 true

### 请假类型
- 生病、发烧、看病、不舒服 → 病假
- 会议、培训、出差、学术活动 → 公假
- 家庭、孩子、私事、婚礼 → 事假
- 其他无法归类 → 其他

### 人工审核判定
以下情况建议人工审核：
- confidence < 0.7
- 日期不确定
- 时段不确定
- 文本和原排班信息存在明显冲突
```

## 节点3：解析结果校验

### 类型设计

| | 输入变量名 | 变量类型 | 必填 | 来源 | 说明 |
|--|-----------|---------|------|------|------|
| **输入** | leave_date | String | 是 | 节点2 | 请假日期 |
| | time_slot | String | 是 | 节点2 | 请假时段 |
| | leave_type | String | 是 | 节点2 | 请假类型 |
| | reason_summary | String | 是 | 节点2 | 原因摘要 |
| | confidence | Number | 是 | 节点2 | 解析置信度 |
| | need_manual_review | Boolean | 是 | 节点2 | 是否建议人工审核 |
| | original_schedule | Object | 是 | 节点1 | 原排班信息 |
| | leave_rules | Object | 是 | 节点1 | 业务规则 |
| **输出** | parsed_leave | Object | - | - | 标准化后的请假信息 |
| | valid | Boolean | - | - | 是否通过基础校验 |
| | errors | Array<String> | - | - | 错误列表 |
| | warnings | Array<String> | - | - | 警告列表 |
| | final_manual_review | Boolean | - | - | 最终是否需要人工审核 |

### 代码

```javascript
//only dayjs and lodash are allowed
import dayjs from 'dayjs';
import _ from 'lodash';

async function main({ params }: Args): Promise<Output> {
  const errors = [];
  const warnings = [];

  const leaveDate = params.leave_date || '';
  const timeSlot = params.time_slot || '';
  const leaveType = params.leave_type || '';
  const reasonSummary = params.reason_summary || '';
  const confidence = Number(params.confidence || 0);
  const needManualReview = !!params.need_manual_review;
  const originalSchedule = params.original_schedule || {};
  const leaveRules = params.leave_rules || {};

  const isValidDateFormat = /^\d{4}-\d{2}-\d{2}$/.test(leaveDate);
  if (!isValidDateFormat || !dayjs(leaveDate).isValid()) {
    errors.push('请假日期格式不合法');
  }

  if (!['上午', '下午', '全天'].includes(timeSlot)) {
    errors.push('请假时段不合法');
  }

  if (!['事假', '病假', '公假', '其他'].includes(leaveType)) {
    errors.push('请假类型不合法');
  }

  if (!String(reasonSummary).trim()) {
    errors.push('请假原因摘要不能为空');
  }

  if (confidence < 0 || confidence > 1) {
    errors.push('置信度必须在0到1之间');
  }

  if (originalSchedule.work_date && leaveDate && originalSchedule.work_date !== leaveDate) {
    warnings.push('解析日期与原排班日期不一致，建议人工复核');
  }

  let finalManualReview = needManualReview;
  if (leaveRules.low_conf_review && confidence < 0.7) {
    finalManualReview = true;
    warnings.push('置信度低于0.7，必须人工审核');
  }

  if (errors.length > 0) {
    finalManualReview = true;
  }

  return {
    parsed_leave: {
      leave_date: leaveDate,
      time_slot: timeSlot,
      leave_type: leaveType,
      reason_summary: reasonSummary,
      confidence: confidence,
      need_manual_review: finalManualReview
    },
    valid: errors.length === 0,
    errors: errors,
    warnings: warnings,
    final_manual_review: finalManualReview
  };
}
```

### 类型设计

| | 输入变量名 | 变量类型 | 必填 | 来源 | 说明 |

| **输入** | rec_substitute | Object | 否 | 节点4 | 推荐替班医生 |

| | adjustment_reason | String | 是 | 节点4 | 调整理由 |
| | risk_level | String | 是 | 节点4 | 风险等级 |
| | need_manual_review | Boolean | 是 | 节点4 | 是否建议人工审核 |
| | parsed_leave | Object | 是 | 节点3 | 请假信息 |

| | errors | Array<String> | - | - | 错误列表 |
| | warnings | Array<String> | - | - | 警告列表 |
| | validated_plan | Object | - | - | 校验通过的最终方案 |
| | final_manual_review | Boolean | - | - | 是否需人工审核 |

### 代码

```javascript
//only dayjs and lodash are allowed
import dayjs from 'dayjs';
import _ from 'lodash';

async function main({ params }: Args): Promise<Output> {
  const errors = [];
  const warnings = [];

  const recommended = params.rec_substitute;
  const quotaTransfer = Number(params.quota_transfer || 0);
  const remainingQuota = Number(params.remaining_quota || 0);
  const affectedPatients = Number(params.affected_patients || 0);
  const adjustmentReason = params.adjustment_reason || '';
  const riskLevel = params.risk_level || '高';
  const needManualReview = !!params.need_manual_review;
  const originalSchedule = params.original_schedule || {};
  const candidates = params.candidates || [];
  const leaveRules = params.leave_rules || {};

  let finalManualReview = needManualReview;
  let matchedCandidate = null;

  if (!['低', '中', '高'].includes(riskLevel)) {
    errors.push('风险等级不合法');
  }

  if (!String(adjustmentReason).trim()) {
    errors.push('调整理由不能为空');
  }

  if (quotaTransfer < 0 || remainingQuota < 0 || affectedPatients < 0) {
    errors.push('数量字段不能小于0');
  }

  const originalAvailable = Number(originalSchedule.available_quota || 0);
  if (quotaTransfer > originalAvailable) {
    errors.push('转移号源数不能超过原排班剩余号源');
  }

  if (recommended) {
    matchedCandidate = candidates.find(item => item.physician_id === recommended.physician_id) || null;

    if (!matchedCandidate) {
      errors.push('推荐替班医生不在候选列表中');
    }
  } else {
    warnings.push('无推荐替班医生，必须人工处理');
    finalManualReview = true;
  }

  if (matchedCandidate) {
    if (quotaTransfer > Number(matchedCandidate.available_quota || 0)) {
      errors.push('替班医生可承接号源不足');
    }

    const totalQuota = Number(matchedCandidate.total_quota || 0);
    const usedQuota = Number(matchedCandidate.used_quota || 0);
    const afterUsage = totalQuota > 0 ? (usedQuota + quotaTransfer) / totalQuota : 1;
    const maxUsage = Number(leaveRules.max_usage_rate || 0.85);

    if (afterUsage > maxUsage) {
      warnings.push('替班后号源利用率超过规则阈值');
      finalManualReview = true;
    }

    const originalTitle = String(originalSchedule.physician_title || '');
    const candidateTitle = String(matchedCandidate.physician_title || '');
    const sameLevel = originalTitle === candidateTitle;

    if (leaveRules.same_level_first && !sameLevel) {
      warnings.push('未命中同级别优先规则，建议人工复核');
      finalManualReview = true;
    }
  }

  if (riskLevel === '高') {
    finalManualReview = true;
  }

  if (errors.length > 0) {
    finalManualReview = true;
  }

  return {
    valid: errors.length === 0,
    errors: errors,
    warnings: warnings,
    validated_plan: {
      rec_substitute: recommended,
      quota_transfer: quotaTransfer,
      remaining_quota: remainingQuota,
      affected_patients: affectedPatients,
      adjustment_reason: adjustmentReason,
      risk_level: riskLevel,
      need_manual_review: finalManualReview
    },
    final_manual_review: finalManualReview
  };
}
```

## 调用方式说明

### 调用原则

本工作流发布后，调用方只做一件事：

1. 按文档准备完整 JSON
2. 调用 Coze 工作流 URL
3. 读取 Coze 返回结果

Coze 不会在处理过程中再去调用任何业务系统方法，也不会自行补齐缺失字段。

### 调用方最少需要具备的能力

严格来说，这里不要求“后端先查库再组装”这套能力写进 Coze 设计里；只要求 **调用 Coze 的那一侧** 最终能够提供以下完整数据：

- physician_id (Integer)
- physician_name (String)
- physician_title (String)
- department_id (Integer)
- department_name (String)
- raw_text (String)
- original_schedule (Object)
- candidates (Array<SubstituteCandidate>)
- leave_rules (Object)

如果缺少其中任一关键字段，Coze 只能基于不完整输入返回低置信度结果，或者触发人工审核。
