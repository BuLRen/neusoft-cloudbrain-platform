## 1. 文档概述

### 1.1 编写目的

本文档是希康云医院智能排班系统的设计文档，用于指导开发团队进行系统实现。智能排班系统采用 **Coze 平台 + 自建服务** 的混合架构，实现 AI 驱动的医院排班管理。

### 1.2 设计目标

| 目标 | 说明 |
|------|------|
| **AI 赋能排班** | 利用 AI 自动生成排班方案、分析号源、推荐替班 |
| **管理员可控** | 所有 AI 生成的方案都需管理员确认后生效 |
| **流程完整** | 覆盖排班生成、请假处理、紧急调整、号源预警全场景 |
| **高效协作** | 支持多角色协作（医生请假、管理员审批、患者通知） |

### 1.3 参考文档

| 文档名称 | 说明 |
| --- | --- |
| 01_系统架构设计文档.md | 微服务架构、AI服务设计 |
| 数据库设计 | doctor_schedule、schedule_plan 等表结构 |

## 3. 微服务框架

### 3.1 服务职责

| 服务 | 端口 | 职责 | 技术栈 |
| --- | --- | --- | --- |
| `schedule-service` | 8095 | 智能排班核心服务 | Spring Boot 3.2 + MyBatis + PostgreSQL |
| Coze 平台 | SaaS | AI 能力（生成、分析、对话） | Coze 工作流 + Bot |

### 3.2 服务间调用关系

```
gateway-service (:8080)
      │
      ├── 路由到 schedule-service (:8095)
      │       │
      │       ├── 挂号收费查询排班、扣减号源
      │       ├── 医生工作站查询排班、申请请假
      │       └── 患者端查询可用排班
      │
      └── 路由到其他业务服务

schedule-service (:8095)
      │
      ├── 被 Coze 平台调用（Webhook 回调）
      ├── 被 registration-service 调用（号源管理）
      ├── 被 physician-service 调用（排班查询）
      └── 发送通知（邮件/短信/站内信）
```

```sql

CREATE TABLE schedule_plan (
    id              SERIAL          PRIMARY KEY,
    plan_name       VARCHAR(64)    NOT NULL,

    status          VARCHAR(16)     NOT NULL DEFAULT '草稿',

);

CREATE INDEX idx_sp_dept_month ON schedule_plan(department_id, plan_month);
CREATE INDEX idx_sp_status ON schedule_plan(status);
```

## 5. 工作流设计

### 5.1 工作流总览

| 工作流 | 触发方式 | 节点数 | 核心功能 |
|--------|----------|--------|----------|
| **WF-01: AI生成月度排班** | 手动/定时 | 7 | AI生成排班方案 → 管理员审核 → 发布 |
| **WF-02: 处理医生请假** | 用户输入 | 8 | 请假 → AI理解 → 生成调整方案 → 管理员确认 |
| **WF-03: 管理员紧急调整** | 手动触发 | 6 | 手动调整 → AI建议 → 预览确认 → 执行 |
| **WF-04: 号源预警和平衡** | 定时/手动 | 7 | 检测号源 → AI分析 → 生成建议 → 通知 |

### 5.2 工作流一：AI生成月度排班

#### 流程图

```
┌───┐   ┌───┐   ┌───┐   ┌───┐   ┌───┐   ┌───┐   ┌───┐
│触发│→ │获取│→ │AI │→ │校验│→ │保存│→ │通知│→ │结束│
│   │  │数据│  │生成│  │   │  │   │  │   │
└───┘  └───┘  └───┘  └───┘  └───┘  └───┘  └───┘
```

#### 节点详细设计

| 节点 | 类型 | 输入 | 输出 | 异常处理 |
|------|------|------|------|----------|
| **1. 触发器** | 开始 | department_id, month, generate_type | 触发信号 | 参数校验失败返回错误 |
| **2. 获取基础数据** | API调用 | department_id | doctors[], history[], holidays[] | 超时使用缓存，异常记录日志 |
| **3. AI生成排班** | LLM | doctors, history, holidays, constraints | schedule_plan JSON | 超时重试1次，还失败返回原始结果 |
| **4. 数据校验** | 代码逻辑 | AI输出 | 校验通过的排班数据 | 失败重试或标记需人工审核 |
| **5. 保存数据库** | API调用 | 排班数据 | plan_id, schedule_ids | 失败事务回滚 |
| **6. 发送通知** | 消息发送 | plan_id, preview | 通知发送结果 | 失败不影响主流程 |
| **7. 输出结果** | 结束 | 执行结果 | plan_id, preview, suggestions | - |

```javascript
/**
 * 排班数据校验与转换
 * 功能：
 * 1. 校验日期格式
 * 2. 校验时段合法性
 * 3. 检测重复排班
 * 4. 转换日期格式
 * 5. 补充默认值
 */

function validateAndTransform(input) {
    const { schedules, statistics } = input;
    const errors = [];
    const validated = [];
    
    // 日期正则表达式
    const dateRegex = /^\d{4}-\d{2}-\d{2}$/;
    
    // 已见排班集合（用于检测重复）
    const seen = new Set();
    
    // 统计医生出勤次数
    const doctorWorkCount = {};
    
    // 遍历每个排班进行校验
    for (let i = 0; i < schedules.length; i++) {
        const schedule = schedules[i];
        const scheduleErrors = [];
        
        // 1. 校验日期格式
        if (!dateRegex.test(schedule.work_date)) {
            scheduleErrors.push(`第${i+1}条：日期格式错误 ${schedule.work_date}`);
        }
        
        // 2. 校验时段
        if (!["上午", "下午", "晚上"].includes(schedule.time_slot)) {
            scheduleErrors.push(`第${i+1}条：时段错误 ${schedule.time_slot}`);
        }
        
        // 3. 校验医生ID
        if (!schedule.physician_id || typeof schedule.physician_id !== 'number') {
            scheduleErrors.push(`第${i+1}条：医生ID无效`);
        }
        
        // 4. 检测重复排班
        const key = `${schedule.work_date}_${schedule.physician_id}_${schedule.time_slot}`;
        if (seen.has(key)) {
            scheduleErrors.push(`第${i+1}条：重复排班 ${schedule.physician_name} ${schedule.work_date} ${schedule.time_slot}`);
        }
        seen.add(key);
        
        // 5. 统计医生出勤次数
        if (!doctorWorkCount[schedule.physician_id]) {
            doctorWorkCount[schedule.physician_id] = 0;
        }
        doctorWorkCount[schedule.physician_id]++;
        
        // 6. 检测医生工作量超限（每周>4次）
        if (doctorWorkCount[schedule.physician_id] > 4) {
            scheduleErrors.push(`第${i+1}条：${schedule.physician_name}周工作量超过4次`);
        }
        
        // 汇总错误
        if (scheduleErrors.length > 0) {
            errors.push(...scheduleErrors);
        } else {
            // 添加默认值
            validated.push({
                physician_id: schedule.physician_id,
                physician_name: schedule.physician_name,
                work_date: schedule.work_date,
                time_slot: schedule.time_slot,
                total_quota: schedule.total_quota || 30,
                ai_suggestion: schedule.ai_suggestion || ""
            });
        }
    }
    
    // 返回结果
    if (errors.length > 0) {
        return {
            valid: false,
            errors: errors,
            validated_schedules: [],
            statistics: {}
        };
    }
    
    // 计算周统计
    const weeklyStats = {};
    for (const [doctorId, count] of Object.entries(doctorWorkCount)) {
        weeklyStats[doctorId] = count;
    }
    
    return {
        valid: true,
        errors: [],
        validated_schedules: validated,
        statistics: {
            ...statistics,
            validated_count: validated.length,
            weekly_stats: weeklyStats
        },
        warnings: validated.length < statistics.total_schedules ? 
            ["部分排班校验失败，已过滤"] : []
    };
}

// 执行校验
const result = validateAndTransform({
    schedules: input.schedules,
    statistics: input.statistics
});

// 输出结果
output.valid = result.valid;
output.errors = result.errors;
output.validated_schedules = result.validated_schedules;
output.statistics = result.statistics;
output.warnings = result.warnings || [];
```

|--------|------|------|

| errors | Array | 错误列表 |
| validated_schedules | Array | 校验通过的排班 |

```markdown
你是一个医院排班系统的自然语言理解（NLU）助手，专门负责从医生提交的请假文本中提取结构化的请假信息。

## 你的任务

分析医生提交的请假文本，提取以下信息：
1. **请假日期**：具体哪一天
2. **请假时段**：上午/下午/全天
3. **请假类型**：事假/病假/公假/其他
4. **请假原因**：简要概括（10字内）

## 解析规则

### 日期解析
- 明确日期（如"6月10日"）→ 转换为 YYYY-MM-DD 格式
- 相对日期（如"下周三"）→ 推算具体日期
- 如果无法确定，标记为需要用户确认

### 时段解析
- 明确时段（如"上午"、"下午"）→ 直接使用
- 未明确 → 根据上下文推断，默认为"上午"
- "全天"表示上下午都请假

### 请假类型判断
- "生病"、"看病"、"医院" → 病假
- "出差"、"会议"、"培训" → 公假
- "孩子"、"家里"、"私事" → 事假
- 无法判断 → 其他

### 置信度评估
- confidence >= 0.8：文本清晰，解析准确
- 0.6 <= confidence < 0.8：文本有一定模糊，标记需要确认
```

**代码逻辑：**

```javascript
/**
 * 替班方案校验
 * 检查：
 * 1. 替班医生是否有足够号源
 * 2. 替班后利用率是否超限
 * 3. 是否存在冲突排班
 */

function validatePlan(plan, substitutes) {
    const errors = [];
    const warnings = [];
    
    if (!plan.recommended_substitute) {
        return { valid: false, errors: ["无合适替班医生"], warnings: [] };
    }
    
    const sub = plan.recommended_substitute;
    
    // 查找替班医生的号源信息
    const subInfo = substitutes.find(s => s.physician_id === sub.physician_id);
    
    if (!subInfo) {
        errors.push("找不到替班医生信息");
        return { valid: false, errors, warnings };
    }
    
    // 校验1：替班医生号源是否足够
    if (plan.quota_transfer > subInfo.available_quota) {
        errors.push(`替班医生号源不足：需要${plan.quota_transfer}个，可用${subInfo.available_quota}个`);
    }
    
    // 校验2：替班后利用率是否超限
    const newUsage = (subInfo.used_quota + plan.quota_transfer) / subInfo.total_quota;
    if (newUsage > 0.85) {
        warnings.push(`替班后利用率${(newUsage * 100).toFixed(0)}%超过85%阈值`);
    }
    
    // 校验3：替班医生是否已有排班冲突
    // （实际应查询数据库，此处为模拟）
    const hasConflict = false; // 假设无冲突
    if (hasConflict) {
        errors.push("替班医生此时段已有排班");
    }
    
    return {
        valid: errors.length === 0,
        errors,
        warnings,
        plan: {
            ...plan,
            validated: errors.length === 0
        }
    };
}

const result = validatePlan(input, input.substitutes);
output.valid = result.valid;
output.errors = result.errors;
output.warnings = result.warnings;
output.plan = result.plan;
```

##### 节点3：号源统计汇总（代码节点）

```javascript
/**
 * 号源数据统计和问题识别
 */
function analyzeQuotas(quotas) {
    const problems = [];
    const stats = {
        total_schedules: quotas.length,
        total_quota: 0,
        total_available: 0,
        avg_usage_rate: 0,
        critical_schedules: [],
        excess_schedules: []
    };
    
    for (const q of quotas) {
        stats.total_quota += q.total_quota;
        stats.total_available += q.available_quota;
        
        // 识别问题
        const usageRate = q.usage_rate;
        const availablePercent = q.available_quota / q.total_quota;
        
        if (q.available_quota === 0) {
            problems.push({
                type: "号源为0",
                schedule_id: q.schedule_id,
                location: `${q.date} ${q.time_slot} ${q.physician_name}`,
                severity: "高",
                details: "无法挂号"
            });
            stats.critical_schedules.push(q.schedule_id);
        } else if (availablePercent < 0.2) {
            problems.push({
                type: "号源紧张",
                schedule_id: q.schedule_id,
                location: `${q.date} ${q.time_slot} ${q.physician_name}`,
                severity: usageRate > 0.9 ? "高" : "中",
                details: `剩余${q.available_quota}个（${(availablePercent * 100).toFixed(0)}%）`
            });
        } else if (availablePercent > 0.8) {
            stats.excess_schedules.push(q.schedule_id);
        }
    }
    
    stats.avg_usage_rate = stats.total_quota > 0 ? 
        (stats.total_quota - stats.total_available) / stats.total_quota : 0;
    
    return {
        stats,
        problems,
        has_high_severity: problems.some(p => p.severity === "高"),
        has_medium_severity: problems.some(p => p.severity === "中")
    };
}

const analysis = analyzeQuotas(input.quotas);
output.stats = analysis.stats;
output.problems = analysis.problems;
output.has_alert = analysis.has_high_severity || analysis.has_medium_severity;
output.severity = analysis.has_high_severity ? "高" : 
                  analysis.has_medium_severity ? "中" : "低";
```
