## 工作流边界

### Coze 负责

- 分析本次调整的影响范围
- 评估风险等级
- 输出注意事项和 AI 建议
- 生成调整前后预览数据

### 后端负责

- 查询排班详情
- 组装 Coze 输入参数
- 调用 Coze 工作流
- 返回分析结果给前端页面
- 管理员确认后执行数据库更新
- 发送通知并记录日志

### 输入完整性约束

- Coze 不具备调用后端补数的能力
- 开始节点中的每个字段都必须由后端在调用前准备完成
- LLM 提示词和代码节点中引用的变量，只能来自开始节点输入或前序节点输出
- 凡是需要在预览中展示、但 Coze 无法自行查询的信息，也必须由后端提前补齐
- 替班场景如需正确展示新医生职称，后端应在 `new_values` 中一并传入 `new_physician_title`

### 不放在 Coze 中的能力

- 不在 Coze 中查询数据库
- 不在 Coze 中等待人工确认
- 不在 Coze 中执行写库操作
- 不在 Coze 中发送通知

## 数据结构定义

### 调整类型 AdjustType

> 当前 WF-03 对齐现有后端能力，聚焦“已有排班”的紧急调整。

| 值 | 说明 | 当前后端对应能力 |
|----|------|------------------|
| stop | 停诊 | `/api/schedule/schedule/{scheduleId}/stop` |
| substitute | 替班 | `/api/schedule/schedule/{scheduleId}` 更新 physicianId |
| quota | 号源调整 | `/api/schedule/schedule/{scheduleId}` 更新 totalQuota |
| cancel | 取消出诊 | `/api/schedule/schedule/{scheduleId}` 更新 status |

> 说明：`add`（加诊）对应的是“新增排班记录”，当前更适合走创建排班接口，不放入本版 WF-03 范围。

### 排班对象 ScheduleInfo

> 该对象由后端组装后传给 Coze，不要求与数据库实体字段同名。

| 属性名 | 类型 | 说明 | 当前来源 |
|--------|------|------|----------|
| id | Integer | 排班ID | `DoctorSchedule.id` |
| physician_id | Integer | 原医生ID | `DoctorSchedule.physicianId` |
| physician_name | String | 原医生姓名 | `DoctorSchedule.physicianName` |
| physician_title | String | 原医生职称 | `DoctorSchedule.physicianTitle` |
| department_id | Integer | 科室ID | `DoctorSchedule.departmentId` |
| department_name | String | 科室名称 | `DoctorSchedule.departmentName` |
| work_date | String | 出诊日期（YYYY-MM-DD） | `DoctorSchedule.workDate` |
| time_slot | String | 时段：上午/下午 | `DoctorSchedule.timeSlot` |
| total_quota | Integer | 总号源 | `DoctorSchedule.totalQuota` |
| used_quota | Integer | 已用号源 | `DoctorSchedule.usedQuota` |
| available_quota | Integer | 剩余号源 | `DoctorSchedule.availableQuota` |
| status | String | 排班状态 | `DoctorSchedule.status` |

### 调整预览对象 Preview

| 属性名 | 类型 | 说明 |
|--------|------|------|
| before | Object | 调整前信息 |
| after | Object | 调整后信息 |
| changes | Array<Object> | 变更项列表 |
| affected_patients | Integer | 受影响患者数 |
| needs_notification | Boolean | 是否建议通知患者 |

## 节点3：生成调整预览

### 类型设计

| | 输入变量名 | 变量类型 | 必填 | 来源 | 说明 |
|--|-----------|---------|------|------|------|
| **输入** | schedule_info | Object | 是 | 节点1 | 原排班信息 |
| | adjust_type | String | 是 | 节点1 | 调整类型 |
| | new_values | Object | 否 | 节点1 | 新值 |
| | impact_level | String | 是 | 节点2 | 影响范围 |
| | risk_level | String | 是 | 节点2 | 风险等级 |
| | patient_count | Integer | 是 | 节点1 | 已挂号患者数 |
| **输出** | preview | Object | - | - | 调整前后预览 |
| | affected_patients | Integer | - | - | 受影响患者数 |
| | needs_notification | Boolean | - | - | 是否建议通知 |
| | changes | Array<Object> | - | - | 变更项列表 |
| | risk_level | String | - | - | 风险等级 |

### 代码

```javascript
//only dayjs and lodash are allowed
import dayjs from 'dayjs';
import _ from 'lodash';

async function main({ params }: Args): Promise<Output> {
  const scheduleInfo = params.schedule_info || {};
  const adjustType = params.adjust_type || '';
  const newValues = params.new_values || {};
  const impactLevel = params.impact_level || 'none';
  const riskLevel = params.risk_level || 'low';
  const patientCount = params.patient_count || 0;

  const weekdayMap = ['周日', '周一', '周二', '周三', '周四', '周五', '周六'];
  const workDate = scheduleInfo.work_date || '';
  const weekday = workDate ? weekdayMap[dayjs(workDate).day()] : '';

  const before = {
    physician: scheduleInfo.physician_name,
    physician_title: scheduleInfo.physician_title,
    date: workDate,
    weekday,
    time_slot: scheduleInfo.time_slot,
    quota: scheduleInfo.total_quota,
    used_quota: scheduleInfo.used_quota,
    available_quota: scheduleInfo.available_quota,
    status: scheduleInfo.status
  };

  const after = {
    ...before,
    physician: before.physician,
    physician_title: before.physician_title
  };

  if (adjustType === 'stop') {
    after.status = '停诊';
  }

  if (adjustType === 'substitute') {
    after.physician = newValues.new_physician_name || before.physician;
    after.physician_title = newValues.new_physician_title || before.physician_title;
  }

  if (adjustType === 'quota') {
    after.quota = newValues.new_quota || before.quota;
    after.available_quota = Math.max((newValues.new_quota || before.quota) - (before.used_quota || 0), 0);
  }

  if (adjustType === 'cancel') {
    after.status = '已取消';
  }

  const changes = [];

  if (before.physician !== after.physician) {
    changes.push({
      field: '医生',
      before: `${before.physician}（${before.physician_title}）`,
      after: `${after.physician}（${after.physician_title}）`
    });
  }

  if (before.quota !== after.quota) {
    changes.push({
      field: '号源',
      before: `${before.quota}个`,
      after: `${after.quota}个`
    });
  }

  if (before.status !== after.status) {
    changes.push({
      field: '状态',
      before: before.status,
      after: after.status
    });
  }

  let affectedPatients = 0;
  if (['stop', 'substitute', 'cancel'].includes(adjustType)) {
    affectedPatients = patientCount;
  } else if (adjustType === 'quota') {
    const reduced = (before.quota || 0) - (after.quota || 0);
    affectedPatients = reduced > 0 ? Math.min(reduced, patientCount) : 0;
  }

  const needsNotification = affectedPatients > 0 || impactLevel === 'some' || impactLevel === 'many';

  return {
    preview: {
      before,
      after,
      changes,
      impact_level: impactLevel,
      risk_level: riskLevel
    },
    affected_patients: affectedPatients,
    needs_notification: needsNotification,
    changes,
    risk_level: riskLevel
  };
}
```

## Coze 搭建步骤

```
1. 创建工作流：wf_admin_adjust_analyze

2. 节点1（开始节点）配置：
   - schedule_id (Integer)
   - adjust_type (String)
   - reason (String)
   - new_values (Object) ← 可选
   - operator_id (Integer)
   - schedule_info (Object)
   - patient_count (Integer)

3. 节点2（LLM节点）：分析调整影响
   模型：通义千问2.5，Temperature：0.3
   输入：schedule_id, adjust_type, reason, new_values, operator_id, schedule_info, patient_count
   输出：impact_level, risk_level, considerations, ai_suggestion, next_steps, alternative_plan

4. 节点3（代码节点）：生成调整预览
   输入：schedule_info, adjust_type, new_values, impact_level, risk_level, patient_count
   代码：复制上面的代码
   输出：preview, affected_patients, needs_notification, changes, risk_level

5. 节点4（结束节点）：
   输出：impact_level, risk_level, considerations, ai_suggestion, next_steps,
        alternative_plan, preview, affected_patients, needs_notification, changes, message

6. 连接节点：
   开始 → 节点2 → 节点3 → 结束
```

| 停诊 | POST | `/api/schedule/schedule/{scheduleId}/stop` | 执行停诊 | 已有 |
| 恢复出诊 | POST | `/api/schedule/schedule/{scheduleId}/resume` | 处理恢复场景 | 已有 |

### 接口职责划分

- `GET /api/schedule/detail/{scheduleId}`：后端查排班原始数据
- `POST /api/coze/admin-adjust/analyze`：只做分析，不写数据库
- `PUT /api/schedule/schedule/{scheduleId}`：管理员确认后执行替班、号源调整、取消出诊
- `POST /api/schedule/schedule/{scheduleId}/stop`：管理员确认后执行停诊

## 后端调用 Coze 示例

```java
@PostMapping("/coze/admin-adjust/analyze")
public Map<String, Object> analyzeAdminAdjust(@RequestBody Map<String, Object> request) {
    Long scheduleId = ((Number) request.get("scheduleId")).longValue();
    String adjustType = (String) request.get("adjustType");
    String reason = (String) request.get("reason");
    Long operatorId = ((Number) request.get("operatorId")).longValue();
    Map<String, Object> newValues = (Map<String, Object>) request.get("newValues");

    DoctorSchedule schedule = doctorScheduleService.getById(scheduleId);
    if (schedule == null) {
        return error("排班不存在");
    }

    Map<String, Object> scheduleInfo = new HashMap<>();
    scheduleInfo.put("id", schedule.getId());
    scheduleInfo.put("physician_id", schedule.getPhysicianId());
    scheduleInfo.put("physician_name", schedule.getPhysicianName());
    scheduleInfo.put("physician_title", schedule.getPhysicianTitle());
    scheduleInfo.put("department_id", schedule.getDepartmentId());
    scheduleInfo.put("department_name", schedule.getDepartmentName());
    scheduleInfo.put("work_date", schedule.getWorkDate() != null ? schedule.getWorkDate().toString() : null);
    scheduleInfo.put("time_slot", schedule.getTimeSlot());
    scheduleInfo.put("total_quota", schedule.getTotalQuota());
    scheduleInfo.put("used_quota", schedule.getUsedQuota());
    scheduleInfo.put("available_quota", schedule.getAvailableQuota());
    scheduleInfo.put("status", schedule.getStatus());

    Integer patientCount = schedule.getUsedQuota();

    Map<String, Object> cozeRequest = new HashMap<>();
    cozeRequest.put("schedule_id", scheduleId);
    cozeRequest.put("adjust_type", adjustType);
    cozeRequest.put("reason", reason);
    cozeRequest.put("new_values", newValues);
    cozeRequest.put("operator_id", operatorId);
    cozeRequest.put("schedule_info", scheduleInfo);
    cozeRequest.put("patient_count", patientCount);

    // 需要在 CozeIntegrationService 中新增 analyzeAdminAdjust 方法
    Map<String, Object> cozeResult = cozeIntegrationService.analyzeAdminAdjust(cozeRequest);
    return success(cozeResult);
}
```

### 对齐当前代码的说明

- 复用现有 `DoctorScheduleService#getById` 获取排班详情
- 复用现有 `DoctorSchedule` 扩展字段：`physicianName`、`physicianTitle`、`departmentName`
- `patient_count` 直接取当前 `usedQuota`
- 当前代码中尚无 `physicianInfo` 聚合接口，因此本版 WF-03 不依赖该对象
- 当前 `CozeIntegrationService` 还没有 `analyzeAdminAdjust` 方法，需要新增
